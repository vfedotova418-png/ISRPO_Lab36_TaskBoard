## Описание

Web-приложение Task Board - интерактивный список задач. Реализовано добавление новых задач, редактирование и удаление старых, пометка задачи как выполненой, фильтрация задач по тому, выполнены они или нет.

## Как запустить

Frontend:

1. Клонировать репозиторий [Github](https://github.com/vfedotova418-png/ISRPO_Lab36_TaskBoard.git)
2. Найти файл `index.html`
3. Запустить через расширение `Live Server`

Backend:

4. В терминале выполнить:

```bash
cd TaskBoardApi/
dotnet run
```

## Главные выводы

- Асинхронные методы (async/await) нужны, чтобы сервер не зависал на долгих операциях. Без них приложение тормозило бы.

- Важно разделять логику приложения, его интерфейса и код, который их связывает, чтобы не перегружать приложение и избежать его медленной работы.

- Код лучше разделять на отдельные модули для читаемости, лёгкости редактирования и тестирования приложения.

- CORS — обязательная настройка. Frontend и backend работают на разных портах. Без app.UseCors("AllowAll") браузер заблокирует запросы из соображений безопасности.

- Нужно экранировать ввод пользователя — функция escapeHtml() заменяет опасные символы (например, < на &lt;). Это защищает от XSS-атак, когда кто-то пытается вставить вредоносный скрипт.
